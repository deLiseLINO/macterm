What's Changed

Session persistence

Your terminal sessions now survive quitting Macterm. Panes are wrapped in zmx sessions that keep running in the background, so when you relaunch, each pane reattaches to its still-live shell — scrollback, running processes, and all.

  • Panes are wrapped in zmx sessions with daemon-side foreground resolution (#127)
  • Sessions persist across quit and reattach on relaunch (#131)
  • Settings now surfaces when zmx is unavailable, with session persistence documented (#133)

Note: quick-terminal sessions remain ephemeral, and sessions don't survive a reboot (the zmx daemon dies with the OS).

Fixes

  • Arrow-key shortcuts now match live keypresses again (#136)
  • Unload Project stops the project's shells again, while keeping the layout (#135)
  • Fixed a title-replay render loop (#134)

Under the hood

  • Benchmark window-state resource usage in CI and compare PRs against main (#129, #132)
  • Silence Swift 6 non-Sendable warnings on AppState's poll-event observers (#130)
  • Isolate UserDefaults under test so runs can't corrupt the developer's app state (#128)
