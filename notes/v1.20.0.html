Macterm 1.20.0

🌐 Remote projects

Open a project that lives on another machine over SSH. Each pane runs a persistent [zmx](https://zmx.sh) session on the host, so your shells, running processes, and scrollback live on the server — not your Mac. Quit Macterm and the sessions keep running; relaunch and every pane reattaches. They even survive a local reboot or a dropped connection.

Add one from Sidebar → + → Remote Machine…, or just type an ssh-style path like devbox:~/dev/api into the command palette. Connection settings come from your ~/.ssh/config; authentication (password, key, or 2FA) happens right in the pane. → [Docs](https://thdxg.github.io/macterm/docs/remote-projects)

⌨️ The macterm CLI

A bundled command-line tool to drive the running app — projects, tabs, panes, and sessions — over a local socket, so scripts and AI agents can orchestrate your terminal. It's on PATH inside every Macterm pane automatically.

macterm tab new --run "npm run dev"
macterm grid 2x2 --run "tail -f log/dev.log"
macterm pane run -- git pull

Every command supports --json for scripting. → [Docs](https://thdxg.github.io/macterm/docs/cli)

📁 Centralized project layouts

Declarative project layouts now live in ~/.config/macterm/projects/, one YAML file per project — matched by path, so they survive across git worktrees and sync cleanly in your dotfiles. The old in-repo .macterm/layout.yaml still works but is deprecated. → [Docs](https://macterm.thdxg.dev/docs/declarative-layouts)
